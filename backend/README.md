# Backend

Здесь подробнее описана работа backend части приложения.

## Сервер

Работа сервера описана в модуле app.  

Сервер принимает на эндпоинт "/resume" POST запрос, который должен хранить в себе данные пользователя о его резюме в формате JSON. В ответ сервер возвращает резюме пользователя в формате pdf.  

Что именно делает сервер при таком запросе (функция getResume):

1. Десериализует JSON в структуру ResumeClassic из модуля latex.
2. Создаёт временную директорую по пути "/backend/pdf_resume" с помощью функции os.MkdirTemp. Пример такой директории лежит по этому же пути.
3. Вызывает функцию GeneratePDF из модуля latex. Эта функция создаёт во временной директории tex файл с резюме пользователя, pdf файл, скомпилированный на основе tex файла, и логи компиляции.
4. Посылает скомпилированный pdf файл пользователю.
5. Удаляет временную директорию.

## latex

Теперь поговорим о модуле latex.

Файл ResumeClassic.go описывает структуру ResumeClassic.

В файле generate.go происходит:
- Инициализация шаблонов из модуля text/template. Шаблоны хранятся в директории "/backend/latex/template".
- Создание tex в файла во временной директории по выбранному шаблону и данным пользователя.
- Компиляция pdf файла из вышеупомянутого tex файла.
